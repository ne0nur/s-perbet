### Task 4: Stores & State

**Objective:** Was speichert jeder Store, wie hängen sie zusammen.

**Inhalt:**
- `matchStore`: matches[], aktuellerSpieltag, selectedTournament, aktivePhase, Cache, Live-Poll, Realtime-Subscription
- `tipStore`: meineTipps[], tippSpeichern()
- `tournamentStore`: tournament_configs[], getTournament(name)
- `authStore`: session, user, signIn, signOut
- Wichtige Querbeziehungen (matchStore.selectedTournament steuert was tipStore lädt)

### Task 5: DB-Schema & Scoring

**Objective:** Tabellen, RLS, Trigger, Punktesystem.

**Inhalt:**
- Tabellen: matches, tips, profiles, leagues, league_members, tournament_configs, seasons, app_settings, chat_messages
- RLS-Regeln: Tipps vor Anpfiff nur für Ersteller sichtbar
- Punktesystem: Distance-based 4-3-2-1-0-−1-−2 (berechnePunkte)
- Trigger: trigger_punkte_bei_ergebnis (ONLY — alter trigger_punkteberechnung gelöscht!)
- Level-System: linear (required_exp = level × 8), server-seitig via update-user-levels Edge Function
- Achievements: 4 Raritäten (Gewöhnlich/Selten/Episch/Legendär), EXP: 50/100/200/500

### Task 6: Sync-Architektur

**Objective:** Wie kommen Live-Daten in die App.

**Inhalt:**
- Primär: S4-Mini (Termux auf Android) ruft Edge Function `sync-match-results` auf
- Edge Function: ESPN API → Matches updaten → KO-Winner propagieren → Heartbeat in app_settings
- Backup: GitHub Actions cron (throttled ~1x/h!)
- Client: 30s Unified Poll (matches + app_settings zusammen)
- Live-Indikatoren: 🟢 syncLabel + letztesUpdate, 🔴 LIVE + spielminute — MÜSSEN synchron sein

### Task 7: i18n & Design-Konventionen

**Objective:** Sprach-System und UI-Regeln.

**Inhalt:**
- 3 Sprachen: de, en, tr — ALLE gleichwertig
- useTranslation() Hook, t('key')
- NIE hardcoded Strings — immer t()
- TOASTS müssen dreisprachig sein
- Design: Dark-Only, Glassmorphism, MD3-Tokens
- Colors: NIE raw Tailwind — primary-container/surface tokens
- Animationen: langsam (~3.2s), asynchron, CSS @keyframes
- UI-Präferenz: Minimal Labels (Zahlen statt Text), kein "Schickimicki"

### Task 8: Kritische Pitfalls

**Objective:** Die teuersten Fehler der Projekt-Historie dokumentieren.

**Inhalt (Top 10):**
1. Doppelte Punkte-Trigger → alter trigger_punkteberechnung + neuer trigger_punkte_bei_ergebnis
2. isFuturePhase null-guard → aktivePhase===null = ALLE Phasen tippbar
3. readOnly-Prop für StandingsPage → keine Tipp-UI in Tabellen-Seite
4. MatchCard statt nackter divs → existierende Komponente nutzen
5. Phase-Label-Duplizierung → getPhaseLabel in 3 Pages, keine Shared-Utility
6. Leaderboard-Cache-Staleness → stale-while-revalidate
7. Live-Indikatoren-Desync → ein Poll, nicht zwei
8. RETURNS TABLE column ambiguity → Tabellen-Alias in PL/pgSQL
9. Achievement-Evaluator data subsets → tipsByMatchday ungefiltert
10. GitHub Actions Throttling → kein Verlass auf Cron
